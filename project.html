Software Specifications

Basic Micro Studio is the main piece of software that is used to write code for any of the Basic Micro products such as BasicATOMs or BasicATOM Pro modules. It is commonly referred to as an Integrated Development Environment or IDE for short. The IDE contains three main parts. 1. A text editor for writing programs 2. A compiler to translate the program into something the microcontroller will understand 3. A loader to download the program to the microcontroller.

Hardware Specifications

The overall specification of the A4WD1 Rover is as follows. Overall Length: 12.00″ Overall Width: 13.50″ Tire Height: 4.75″ Chassis Length: 9.75″ Chassis Width: 8.00″ Chassis Height: 4.00″ Ground Clearance: 1.63″ Weight: 4lbs 6 oz. Speed: 36″ per second.

The A4WD1 Rover consists of following hardware. 1. A4WD1 Rover Body Kit 2. Bot Board II 3. The BASIC Atom 28 Pin 4. Saber tooth V1.03(SSC-32 Servo Controller) 5. The DB9 Serial Data Cable – 6′ 6. Servo Motors 7. Off Road Robot Tire – 4.75″D x 2.375″W (pair) 8. 12mm Hex Mounting Hub – 6mm shaft (pair) 9. 12.0 Volt Ni-MH 2800mAh Battery Pack 10. 6.0 – 12vdc Ni-CD & Ni-MH Universal Smart Charger 11. Wiring Harness – Battery Connector

Conclusion

In this project, we successfully assembled, programmed and implemented an autonomous A4WD1 v2 robot. The rover moves forward, backward, left, or right, depending on the direction in which the rover finds the obstacle. During the robot's implementation, classmates worked effectively to achieve the goal of testing a 4WD rover with two front and one rear sensors. Later, you can use the robot for various purposes such as object detection, image capture, and espionage. However, there are restrictions such as sensors. If the rover is fast, the rover will continue to move, causing a collision. Autonomous rover also has no sensor at the bottom of the chassis. Therefore, it is not possible to detect obstacles under the rover. Also, there are no sensors or programs to prevent it from falling. For future work, we plan to add sensors that detect all obstacles that get in the way. Also, I would like to install a wireless camera and see the actual scenery from a distance. When analyzing cameras and sensors, I would like to program them to prevent crashes. In the future, I would like to place GPS on the rover, perform ground-based checks, and control the rover from the base. Another plan is to add hardware to the existing 4x4 autonomous rover for advanced applications. B. A robot arm to grab something.
